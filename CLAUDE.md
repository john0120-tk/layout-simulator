# Layout Simulator - CLAUDE.md

## プロジェクト概要

キャンプのテント・ギアを購入前に仮想配置して、サイトのスペース感を確認するツール。
2Dの平面上にアイテムを実寸サイズで自由に配置できるレイアウトシミュレーター。
Webブラウザ動作、PCメイン・スマホ対応のレスポンシブデザイン。

## 要件定義（確定）

### コアコンセプト
「テント・ギアを購入前に仮想配置して、キャンプサイトのスペース感を確認するツール」

### 機能要件

| 優先度 | 機能 |
|--------|------|
| Critical | ドラッグ&ドロップで配置・移動 |
| Critical | アイテムの回転 |
| Critical | 実寸サイズ（cm単位）の設定 |
| Critical | 画像エクスポート |
| Critical | フィールドサイズの自由設定 |
| Critical | ユーザー定義アイテム（形状・名前・サイズ） |
| Critical | レイヤー構造（フィールド > テント > ギア） |
| High | 四角形 + 三角形〜六角形の多角形対応 |
| High | 寸法ラベルの表示切替 |
| High | 複数レイアウト管理 |
| Nice to Have | リサイズ（配置後） |
| Nice to Have | グリッドスナップ（自由/スナップ切替） |
| Nice to Have | アイテム間の距離計測ツール |
| Nice to Have | グリッド目盛り表示 |
| Nice to Have | URLシェア・ファイルエクスポート/インポート |
| Nice to Have | カスタム多角形 |
| Nice to Have | ログイン機能（なしでも動作） |

### 非機能要件
- PCブラウザがメインターゲット（スマホも動作保証）
- スマホ：ピンチズーム・タップ選択・モード切替UI
- ログイン不要で基本機能をすべて利用可能
- プリセットアイテムなし（UIのシンプルさ優先）
- 単位は cm 固定

## 開発ガイドライン

### 計画フェーズのルール

<important>
- 実装前に必ずプランモードで設計を確認する
- 大きな変更はフェーズ分けしたゲート式プランを作成する
- 不明点は AskUserQuestion ツールで確認してから進める
</important>

### コーディングルール

- シンプルさを優先：1つの関数に1つの責務
- 投機的な抽象化をしない（実際に必要になるまで汎用化しない）
- エラーハンドリングはシステム境界（ユーザー入力・外部API）のみ
- コメントはロジックが自明でない箇所にのみ追加

### Git コミットルール

<important>
- ファイル単位で個別コミットを作成する（複数ファイルを1コミットにまとめない）
- コミットメッセージは変更の「なぜ」を説明する
- 作業ブランチ: `claude/layout-simulator-requirements-cJdxY`
- main ブランチには直接プッシュしない
</important>

### コンテキスト管理

- コンテキスト使用率50%で `/compact` を実行する
- CLAUDE.md は200行以内に収める
- 大きなルールセットは `.claude/rules/` に分割する
